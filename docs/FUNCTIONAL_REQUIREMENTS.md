# Функциональные Требования (Functional Requirements)

Данный документ описывает возможности системы для различных ролей и основные бизнес-процессы.

## 1. Роли Пользователей (User Roles)

### Участник (User)
Основной пользователь системы, проходящий олимпиады.
- **Индивидуальный Dashboard**: Просмотр доступных олимпиад с обратным отсчетом до начала.
- **Интерфейс прохождения**: 
  - Просмотр задач (текст, изображения, Markdown).
  - Отправка ответов (автосохранение в `Submission`).
  - **Интерактивная консоль отладки (`PythonShell`)**: отладка и локальный запуск Python-кода непосредственно на странице задачи с выводом потока stdout и ошибок.
  - Таймер реального времени, синхронизированный с сервером.
  - **Анти-чит защита**: Ограничение копирования, вставки и детекция ухода со страницы (blur).
- **Результаты**: Просмотр набранных баллов и итогового рейтинга после завершения работы и окончания ручной проверки.

### Администратор (Admin)
Пользователь с полным доступом к управлению системой.
- **Конструктор Олимпиад**: Создание, редактирование и удаление олимпиад и задач.
- **Управление пользователями**: Редактирование ролей, меток (labels) и групп.
- **Мониторинг и Проверка**: 
  - Просмотр аналитики по олимпиадам и ответов участников в реальном времени.
  - **Ручная проверка (Grading)**: Удобный интерфейс ручного оценивания (`ManualGrading`) ответов типа "Текст" и "Код", позволяющий корректировать баллы и оставлять фидбек.
  - **Интерактивный запуск решений (`StudentCodeRunner`)**: Запуск кода участников на любых тестовых входных данных (`stdin`), включая произвольный ручной ввод, и сопоставление вывода (`stdout`) с эталонами.
  - **Контроль нарушений**: Просмотр логов подозрительной активности и ручное аннулирование результатов.
- **Системные настройки**: Управление регистрацией, создание бэкапов.

## 2. Ключевые Возможности (Key Features)

### Конструктор Задач (Task Builder)
- Поддержка форматированного текста (Markdown).
- Установка баллов за каждую конкретную задачу.
- Группировка задач по олимпиадам.
- **Модульная структура конструктора (Test Editor)**: Раздельное редактирование классических вопросов с выбором ответов (`ChoiceEditor`) и продвинутых задач по программированию (`CodeEditor`) с настройкой тест-кейсов, шаблонов кодов и лимитов выполнения.
- **Smart-настройки**: Перемешивание задач (`shuffleTasks`) для каждого участника.

### Система Анти-чита (Anti-Cheat)
- **Блокировка Copy-Paste**: Запрет использования буфера обмена на странице теста.
- **Детекция фокуса**: Логирование событий ухода пользователя со вкладки браузера.
- **Журнал нарушений**: Автоматическая запись инцидентов в БД для последующего анализа администратором.

### Синхронизация Времени
Система строго отслеживает время начала и конца олимпиады на сервере. При истечении времени доступ к редактированию ответов автоматически блокируется через Middleware и Server Actions.

### Гибридная Система Оценки
- **Автоматическая**: Мгновенная проверка тестов и числовых ответов.
- **Ручная**: Проверка развернутых текстовых ответов и программного кода через специализированную панель оценивания с поддержкой перерасчета итогового рейтинга.
- **Ранер решений участников**: Возможность для проверяющего запускать и отлаживать присланный код в изолированном окружении.
- **Рейтинг**: Итоговый балл обновляется динамически в процессе проверки.

## 3. Нефункциональные Требования (Non-functional Requirements)

- **Безопасность**: Защита от SQL-инъекций и CSRF через использование Prisma и Next.js Server Actions.
- **Производительность**: Использование Server-Side Rendering (SSR) для мгновенной загрузки страниц.
- **Адаптивность**: Полная поддержка мобильных устройств для прохождения олимпиад "на ходу".
- **Отказоустойчивость**: Автоматический перезапуск контейнеров Docker при сбоях.

## 4. Сценарии Использования (Use Cases)

1. **Регистрация на олимпиаду**: Пользователь заходит в Dashboard, выбирает активную олимпиаду и нажимает "Начать". Создается запись в `Result`.
2. **Отправка ответа**: При вводе ответа в поле задачи, данные отправляются на сервер и сохраняются в `Submission`. Это гарантирует сохранение прогресса при обрыве связи.
3. **Завершение**: Пользователь нажимает "Завершить работу", система фиксирует `finishedAt` и закрывает доступ к редактированию.
