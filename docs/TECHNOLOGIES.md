## 1. Обзор Технологического Стека

Проект построен на максимально современном и стабильном стеке технологий для обеспечения долгосрочной поддержки и высокой производительности.

## Core Frameworks

### [Next.js 15.1+](https://nextjs.org/)
- **Почему**: Использование App Router позволяет эффективно разделять Server и Client Components, минимизируя размер JS бандла на клиенте.
- **Особенности**: Активное использование Server Actions для всех мутаций данных (Zero-API approach).

### [React 19](https://react.dev/)
- **Почему**: Использование новых хуков (`useActionState`, `useFormStatus`) для упрощения работы с формами и серверными экшенами.

### [Pyodide (WebAssembly)](https://pyodide.org/)
- **Почему**: Позволяет выполнять Python код непосредственно в браузере. Это обеспечивает мгновенную обратную связь для участника без нагрузки на серверную инфраструктуру.
- **Web Workers**: Изоляция выполнения кода в отдельном потоке гарантирует стабильность основного UI приложения.

### [Python 3.x (Server-Side)](https://www.python.org/)
- **Почему**: Используется для финальной оценки решений участников на стороне бэкенда в контейнере приложения.
- **Особенности**: Запуск через `child_process` с ограничением памяти (`resource.setrlimit`), переопределением импортов для безопасности и криптографической валидацией успешного прохождения тестов.

## Data Layer

### [PostgreSQL 15+](https://www.postgresql.org/)
- **Почему**: Стандарт индустрии для реляционных данных с отличной поддержкой JSONB.
- **Storage**: Хранится в Docker Volume для персистентности.

### [Prisma ORM (v6+)](https://www.prisma.io/)
- **Почему**: Автоматическая генерация типов TypeScript на основе схемы БД. Это исключает ошибки несоответствия типов данных между БД и кодом приложения.
- **Миграции**: Управление схемой через `prisma migrate`.

---

## Auth & Security

### [NextAuth.js v5 (Auth.js)](https://authjs.dev/)
- **Почему**: Полная интеграция с Next.js Middleware и Server Components.
- **Credentials Provider**: Реализована безопасная авторизация через email/password.

### [Bcrypt (Native)](https://github.com/kelektiv/node.bcrypt.js)
- **Почему**: В отличие от JS-реализаций, нативный бинарный модуль работает в десятки раз быстрее и безопаснее.

## UI & UX

### [Tailwind CSS v4](https://tailwindcss.com/)
- **Почему**: Позволяет создавать уникальный дизайн без написания громоздких CSS файлов.

### [Shadcn UI](https://ui.shadcn.com/)
- **Почему**: Коллекция высококачественных компонентов. Мы не зависим от библиотеки как "черного ящика", а полностью контролируем код компонентов.

### [Framer Motion](https://www.framer.com/motion/)
- **Почему**: Обеспечивает плавные анимации и микро-интерактивы.

## Инфраструктура и Тесты

### [Docker / Docker Compose](https://www.docker.com/)
- **Почему**: Гарантия того, что приложение запустится на любом сервере абсолютно одинаково.

### [Playwright](https://playwright.dev/)
- **Почему**: Лучший инструмент для E2E тестирования. Имитирует реальное поведение пользователя в браузере.

### [Cloudflare](https://www.cloudflare.com/)
- **Почему**: Обеспечивает защиту от DDOS, управление DNS и ускорение доставки контента (CDN).
