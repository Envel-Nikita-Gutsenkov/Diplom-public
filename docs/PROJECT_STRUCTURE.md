# Обзор структуры проекта (Diplom-public)

Проект организован по современным стандартам Next.js с четким разделением ответственности между слоями (UI, Бизнес-логика, Данные).

## Ключевые разделы

### 1. `app/actions/` (Бизнес-логика)
Это **самая важная часть** проекта. Все изменения данных (мутации) проходят через Server Actions. Они обеспечивают:
- **Безопасность**: Проверка прав и ролей (`ensureAdmin`) непосредственно на сервере перед любым действием.
- **Типобезопасность**: Полная интеграция с Prisma с автогенерацией типов.
- **Атомарность мутаций**: Гарантия целостности данных при обновлении оценок или статусов.

### 2. `components/admin/` (Панель управления)
- `test-editor/`: Модульный конструктор задач, разделенный по принципам SOLID на отдельные компоненты:
  - `ChoiceEditor.tsx` — управление классическими вопросами с вариантами ответов.
  - `CodeEditor.tsx` — специализированный редактор задач на программирование (тест-кейсы, лимиты времени/памяти, шаблоны кода).
  - `QuestionCard.tsx` — карточка представления вопроса в конструкторе олимпиады.
- `ManualGrading.tsx`: Панель ручной проверки ответов и решений по программированию с возможностью переоценки результатов.
- `StudentCodeRunner.tsx`: Инструмент отладки и запуска кода участника администратором на встроенных или кастомных данных (`stdin`).

### 3. `components/tasks/` (Интерфейс решения)
- `PythonShell.tsx`: Интерактивная консоль отладки, позволяющая участникам писать, тестировать и отлаживать Python-код прямо на странице задачи.

### 4. `lib/grading/` (Движок оценивания)
- `code-executor.ts`: Отвечает за сборку окружения, передачу стандартных потоков (`stdin`, `stdout`) и координацию выполнения кода во внешнем Web Worker.

### 5. `public/workers/`
- `python-worker.js`: Фоновый Web Worker для выполнения `Pyodide` (WebAssembly). Изолирует выполнение кода в отдельном потоке ОС, гарантируя, что бесконечные циклы или зависания в коде студента не приведут к блокировке или падению основного интерфейса олимпиады.

## 🛠️ Файлы конфигурации

- `.env` / `.env.example`: Переменные окружения, секреты сессий и параметры подключения к БД.
- `tailwind.config.ts` / `postcss.config.mjs`: Конфигурация стилей, шрифтов и тем (включая расширенные палитры для эффектов Glassmorphism).
- `components.json`: Конфигурация дизайн-системы Shadcn UI.
- `next.config.ts`: Безопасность (CSP-заголовки), оптимизация сборки (`output: 'standalone'`).
